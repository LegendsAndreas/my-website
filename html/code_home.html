This was my first program of a decent size, and it all started from a meme about a newbie coding chess in Python (see first picture). I was so pissed off by it, because there was clearly a better way of doing it, that i decided to put my money where my mouth was and made a chess game of my own!

The second image shows what the chessboard looks like in the Windows 11 terminal. I have no clue why the white pawns are purple, it does not show this in my IDE terminal, it might be something with Windows 11. Anyway, you play this in the same way as any chess game, my writing two chessboard squares, the first one having a chess piece. I have also programmed the rules of each chess piece, so that a player cant play it wrong. The hardest piece to program, was actually the pawn piece. The reason being, the way it attacks a chesspiece, is not the same as how it moves. The pawn moves in a straight line, but attacks diagnoally. On top of that, if it is the pawns first move, it can move two chessquares and not just one. This is in contrary to all the other chesspieces, which attacks in the same way that they normally move and is consistent in the way that they move. The horse piece is a little different too, in the sence that it can jump over other chesspieces, however this was a lot easier to program.

An example of a move, would be the player enters "b2" and "b4", which indicates that the player wants to move a piece on b2, to b4. The game then checks the type of chesspiece and checks if that move is legal.

Oh dear GOD this was such a pain in my ass to get working.

So, to set the stage: I was at my school and got to work on their Arduinos, they wanted me to make it measure the temperature for each room they had. Now, their Arduinos already DID measure the temperature, however, they did not always work correctly and so they wanted a new more reliable program. Unforturnatly for me, there was NO documentaton of how the Arduinos work. Nothing about how the code works, nothing about the specific Arduino in use, nothing about how to write code to the Arduino, NOTHING!

Now, if the Arduino board was a more conventional board and more widely used, it would propably not had been that bad of an experience. BUT, the board that I had to configure, was a Clone, which is basically not an officially supported board. Which means, that the process is not always as straight forward to configure it.

The first step was pretty simple, i just had to install the Arduino IDE, which allows me to create and push the code to the Arduino. But before that, I have to figure out what type of board that I am working with, before i can send my code to the board. Now here's the fucky part of the board. The board I had to configure, was technically a combination of TWO Ardunio boards. The first part of the board is made of an Arduino Uno board and the second part is made up of a Board with an ESP8266 chip, which meant that I had to use two different configuration, for each board part.
The first part of the board, is made of an Arduino Uno board.

I often get into situations where i want to log an error, so that i can see what exactly went wrong, along with a list of errors. So, instead of re-writing a code that does that for every new program that i make, i decided to create a GitHub library, that i, along with anyone else, can get every time i need it! In Go, it's also VERY easy to clone a GitHub repository to a Go program, with the "go get" function.

With the program, you can log with 3 options: Info, Warning and Error, depending on the type of error. The program will automatically create a .txt file, that by default is named "logName" and creates it with the user defined path. The program also comes with the function "GetPath", which gets the program execution path, in case you just want it to be saved in the execution directory.

These two programs purpose is the same and nearly functions in the same way. The first one that i created was in Go (go_get_temperature!) and the second one i created, was in Python. The reason there are two of them, is because i made the first program in Go, which is my personal favorite language, and then the school that i go to created a site where we can download the measured temperatures. The site was written with Python as its Backend, which meant that i had to remake the first program, so that it would more easily integrate with the site.

The programs get the temperatures from an MQTT broker, which functions as a medium between the Arduinos and the actual program. Along with the temperature, they also get the time that that temperature was recorded. They get this data in the form of a JSON file, where the data is then extracted and put into an Excel file.
The difference between the Go and Python version, is that Go creates a new Excel file for every day, while the Python program only creates a new Excel file every month and creates a new Excel sheet, for every day.
